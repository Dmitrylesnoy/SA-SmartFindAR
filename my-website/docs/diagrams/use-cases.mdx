# UML Use Cases

## Диаграмма вариантов использования

```pumld
@startuml
left to right direction
skinparam packageStyle rectangle

actor "Владелец предмета" as Owner
actor "Друг" as Friend

rectangle "SmartFind" {
    usecase "LS-1: Просмотр списка\nдоступных вещей" as UC_List
    usecase "Просмотр карточки предмета" as UC_Card
    usecase "Просмотр места предмета на карте" as UC_ShowMap
    usecase "SH-1: Изменение фотографии места предмета\nв общем доступе" as UC_EditShare
    usecase "ED-1: Изменение данных предмета" as UC_Edit
    usecase "Управление доступом\n(поделиться вещью)" as UC_Access
}

UC_Card ..> UC_List : <<include>>
UC_Edit <.. UC_Card : <<extend>>
UC_Access <.. UC_Card : <<extend>>

Owner <|-- Friend

Owner -- UC_Edit
Owner -- UC_Access

Friend -- UC_EditShare
Friend -- UC_ShowMap

UC_EditShare ..> UC_Card : <<include>>
UC_ShowMap ..> UC_Card : <<include>>

note bottom of UC_Access
  Только владелец имеет 
  право управлять доступом
end note

@enduml
```
---

## Описание

Диаграмма вариантов использования (Use Case Diagram) отображает:

- **Акторов** — пользователей и внешние системы, взаимодействующие с продуктом
- **Варианты использования** — функциональные требования к системе
- **Связи между акторами и вариантами использования**
