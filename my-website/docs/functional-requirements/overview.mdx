# 📋 Функциональные требования

## Общая информация

В этом разделе представлены функциональные требования к продукту SmartFindAR, описанные в формате Use Cases.

Каждое требование включает:

- **ID** — уникальный идентификатор
- **Название** — краткое описание функциональности
- **Акторы** — участники взаимодействия
- **Предусловия** — условия для выполнения
- **Основной поток** — успешный сценарий
- **Постусловия** — состояния после выполнения
- **Альтернативные и исключительные потоки**
- **Бизнес правила**

## Список функциональных требований

| ID | Название |
|:---:|:----------|
| CR-1 | Создание и сохранение предмета в хранилище |
| SC-1 | 3D сканирование помещения |
| ED-1 | Изменение фотографии места предмета |
| SH-1 | Изменение предмета в общем доступе |
| LS-1 | Просмотр списка доступных вещей |
