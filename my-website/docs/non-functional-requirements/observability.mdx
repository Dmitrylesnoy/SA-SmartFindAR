# Наблюдаемость

## Аналитика использования

| Свойство | Значение |
|:---------|:----------|
| **ID** | OBS-01 |
| **Название** | Аналитика использования |
| **Приоритет** | Высокий |

**Описание:** Приложение должно собирать обезличенные данные для расчета KPI: время поиска предмета, частота использования, точность позиционирования.

**Обоснование:** Без этих данных невозможно измерить достижение бизнес-целей (MAU, Retention, время поиска).

---

## Мониторинг ошибок

| Свойство | Значение |
|:---------|:----------|
| **ID** | OBS-02 |
| **Название** | Мониторинг ошибок |
| **Приоритет** | Высокий |

**Описание:** В приложение должна быть встроена система сбора логов о падениях с фильтрацией по версии и модели устройства.

**Обоснование:** Позволит команде быстро реагировать на проблемы с конкретными моделями телефонов (риск аппаратных ограничений).
